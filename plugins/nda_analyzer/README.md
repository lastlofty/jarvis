# Плагин nda_analyzer — анализ NDA и маршрутизация

AI-ассистент для анализа NDA/договоров и маршрутизации документов по категориям.

## Скиллы
- `read_nda(path)` — прочитать документ (txt/pdf)
- `route_document(path, category)` — переместить документ в папку категории
  (`<SAFE_ROOT>/documents/<category>`)

## Что проверять в NDA (чек-лист для агента)
1. Стороны договора (кто раскрывает, кто получает информацию)
2. Предмет — что считается конфиденциальной информацией
3. Срок действия NDA и срок обязательств после расторжения
4. Исключения (что НЕ является конфиденциальным)
5. Разрешённое использование и круг лиц с доступом
6. Ответственность за нарушение, неустойка
7. Применимое право и подсудность
8. Возврат/уничтожение информации по требованию

## Логика маршрутизации (пример)
- корректный, готов к подписи → категория `на подпись`
- есть риски/правки → `на доработку`
- не NDA / нерелевантный → `прочее`

## Пример команды
«Прочитай nda.pdf, проверь по чек-листу и отправь в нужную категорию»
