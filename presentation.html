# 02 // если совсем просто

Что это такое простыми словами

Представь голосового помощника, как Алиса или Siri — но он:

  • живёт на твоём компьютере — и реально им управляет: создаёт файлы, открывает программы, делает скриншоты
  • можно бесконечно обучать — кидаешь документы или пишешь «плагин», и у него появляется новый навык
  • понимает и говорит голосом — слушает микрофон и озвучивает ответ
  • думает, как DeepSeek — сначала рассуждает «про себя», потом отвечает

# 04 // главное: как работает диалог

Что происходит за один запрос

1
Ты пишешь или говоришь команду — «какая погода в Москве?»
2
RAG ищет полезные знания в базе и добавляет их в подсказку модели
3
Нейросеть (GLM) понимает запрос и решает: нужен инструмент get_weather
4
Агент выполняет инструмент — реально запрашивает погоду / создаёт файл / ищет в сети
5
Результат возвращается модели → она формулирует понятный ответ
6
Ответ печатается по словам (стриминг) и при желании озвучивается

# 06 // мозг

Мультимодельность

Все модели спрятаны за единым интерфейсом провайдера. Оркестратору всё равно, кто отвечает — можно менять модель, не трогая остальной код.

  • GLM — основная, бесплатная, доступна в РФ
  • Gemini — Google (гео-ограничен)
  • Ollama — локально, офлайн, без интернета
.env
# выбор модели — одна строка LLM_PROVIDER=glm GLM_MODEL=glm-4-flash GLM_THINKING=true # 🧠
# 07 // фишка

💭 Глубокое мышление

Как у DeepSeek и Claude: модель сначала рассуждает «про себя», разбивая задачу на шаги, и только потом выдаёт чистый ответ. Рассуждения видны отдельным приглушённым блоком.

jarvis · deep-thinking
🧠 размышления: Это математическая задача. Разберём по шагам: было 3 яблока, отдала половину = 1.5, осталось 1.5, купила +5 → 6.5 ... Ответ: У Маши стало 6.5 яблок.
# 08 // память и обучение

Как Jarvis «учится»

📚

RAG — база знаний

Кидаешь .txt/.md/.pdf-тексты → агент режет их на кусочки, индексирует и при ответе ищет самое релевантное. Отвечает по твоим документам, а не выдумывает.

🧩

Плагины и скиллы

Плагин = папка с навыком. Пишешь функцию + описание → агент получает новый инструмент. Так добавлены погода, веб-поиск, генерация картинок, зрение.

🔌

MCP

Стандарт подключения внешних «серверов навыков». Jarvis работает как MCP-клиент — можно цеплять готовые инструменты от других.

🛠

Function calling

Модель сама решает, какой инструмент и с какими параметрами вызвать. Агент выполняет и возвращает результат обратно модели.

# 10 // безопасность

Безопасность по умолчанию

  • SAFE_ROOT — все операции с файлами разрешены только в заданной папке (по умолч. рабочий стол). Системные файлы недоступны.
  • Подтверждение удаления — перед удалением агент обязан спросить пользователя.
  • Защита от выхода за пределы — блокируется обход через «..» (path traversal).
  • Ключи в .env — API-ключи не зашиты в код, лежат отдельно и не попадают в репозиторий.
