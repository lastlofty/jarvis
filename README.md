# 🤖 Jarvis — персональный AI-агент для Windows

Jarvis — портативный AI-агент, который понимает команды на русском и английском, управляет файлами / окнами / клавиатурой, отслеживает время в приложениях и принимает команды с мобильного телефона через защищённый WebSocket.

**Выбор модели:** GLM (основная, бесплатная), Gemini или Ollama (локально) — переключается одним параметром.
**Расширяемость:** плагины со скиллами, база знаний RAG и подключение внешних MCP-серверов.

> Этот репозиторий — **ядро** проекта. Flutter-приложение, долговременная память с эмбеддингами и WebRTC — в `docs/ROADMAP.md`.

## 🚀 Быстрый старт (10 минут)

### 1. Клонировать и установить зависимости

```bash
git clone <your-repo-url> jarvis
cd jarvis
python -m venv .venv

# Windows
.venv\Scripts\activate
# Linux/macOS
source .venv/bin/activate

pip install -r requirements.txt
pip install -e .
```

### 2. Получить API-ключ модели (бесплатно)

**GLM (основная, рекомендуется):**
1. Зарегистрироваться на <https://open.bigmodel.cn/>.
2. Создать API-ключ.
3. Модель `glm-4-flash` бесплатна и поддерживает function calling.

**Gemini (альтернатива):** ключ на <https://aistudio.google.com/app/apikey>.
> ⚠️ Gemini API недоступен в некоторых регионах (ошибка `User location is not supported`). В этом случае используйте GLM или Ollama.

**Ollama (локально, без ключа):** установите <https://ollama.com>, затем `ollama pull qwen2.5:7b`.

### 3. Настроить .env

```bash
cp .env.example .env
# Открыть в редакторе и вписать:
#   LLM_PROVIDER=glm           # glm | gemini | ollama
#   GLM_API_KEY=...            # ключ с open.bigmodel.cn
#   AUTH_TOKEN=<сгенерируйте: python -c "import secrets; print(secrets.token_urlsafe(32))">
```

### 4. Запустить

**Консольный режим (без мобильника, для отладки):**

```bash
python -m jarvis.console
```

```
Вы: создай папку проекты
Jarvis: ✅ Папка создана: C:\Users\...\Рабочий стол\проекты

Вы: сколько времени я провёл в браузере сегодня?
Jarvis: За последние 24 часа в chrome.exe — 3.4 часа, 47 сессий.
```

**Серверный режим (для мобильного приложения):**

```bash
python -m jarvis.main
# Запустится FastAPI на 0.0.0.0:8000
```

Проверить health:
```bash
curl http://localhost:8000/health
# {"status":"ok","version":"0.1.0"}
```

Отправить команду:
```bash
curl -X POST http://localhost:8000/command \
  -H "Authorization: Bearer <AUTH_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{"text":"создай папку test"}'
```

## 📡 Удалённое подключение через Tailscale

### Зачем
Tailscale создаёт зашифрованный mesh-VPN между всеми устройствами одного аккаунта без необходимости пробрасывать порты на роутере. Бесплатный план — 100 устройств.

### Настройка

1. Зарегистрироваться на <https://tailscale.com> (бесплатно).
2. На ПК: скачать Windows-клиент, установить, войти.
3. На телефоне: установить Tailscale из App Store / Play Store, войти под тем же аккаунтом.
4. Узнать Tailscale-IP компьютера (что-то вроде `100.85.42.10`):
   ```powershell
   tailscale ip -4
   ```
5. С телефона теперь доступен `http://100.85.42.10:8000`.

> ⚠️ Откройте порт 8000 в Windows Firewall: `netsh advfirewall firewall add rule name="Jarvis" dir=in action=allow protocol=TCP localport=8000`.

## 🧠 Выбор модели

Все модели приведены к единому интерфейсу провайдера (`src/jarvis/llm/providers/`).
Переключение — параметром `LLM_PROVIDER` в `.env` или в настройках GUI:

| Провайдер | Модель по умолчанию | Особенности |
|---|---|---|
| `glm` | `glm-4-flash` | основная, бесплатная, OpenAI-совместимый API, function calling |
| `gemini` | `gemini-1.5-flash` | бесплатный тариф, но гео-ограничения в части стран |
| `ollama` | `qwen2.5:7b` | локально/офлайн, без ключа, нужна модель с поддержкой tools |

**💭 Глубокое мышление (reasoning).** `GLM_THINKING=true` (или тумблер 🧠 в
настройках) включает режим рассуждений: модель сначала «думает про себя»
(стримится отдельным приглушённым блоком, как у DeepSeek), затем даёт ответ.
Использует бесплатную reasoning-модель `glm-4.5-flash` (`GLM_THINKING_MODEL`).

## 🧩 Плагины, скиллы и RAG

«MCP из плагинов из скиллов + RAG» — расширяем способности агента и обучаем его.

**Плагин** — папка в `plugins/`:

```
plugins/<name>/
    manifest.json     # {name, description, knowledge?: [...], enabled?: true}
    skill.py          # опц.: DECLARATIONS (схемы) + HANDLERS (реализации)
    *.md / *.txt      # опц.: документы для базы знаний (RAG)
```

- **Скиллы** — новые инструменты для модели. В `skill.py` объявляются `DECLARATIONS`
  и `HANDLERS`, загрузчик регистрирует их автоматически.
- **Знания (RAG)** — `.md`/`.txt` индексируются в базу знаний; перед ответом агент
  ищет релевантные фрагменты и подмешивает их в контекст (`RAG_ENABLED=true`).
- Встроенные RAG-инструменты: `rag_search`, `rag_index`, `rag_stats`.
- **Семантический поиск**: `RAG_EMBEDDINGS=true` переключает поиск с TF-IDF на
  эмбеддинги GLM (`embedding-3`) — точнее, но требует API-вызовов.

## 🧠 Долговременная память

Агент помнит факты **между сессиями**. Инструменты `remember` (сохранить факт)
и `recall` (вспомнить); релевантные воспоминания автоматически подмешиваются в
контекст. Хранится в `MEMORY_DB_PATH`. Пример: «запомни, что я люблю тёмную тему» →
в следующем запуске агент это учитывает.

## 🗒 Автономный планировщик

Для сложных многошаговых задач агент сначала вызывает `create_plan`, затем
выполняет шаги обычными инструментами и отмечает прогресс через `complete_step`.
Получается прозрачный цикл *план → действие → отметка*.

## 🔔 Трей, хоткей и wake-word

- Окно сворачивается в **системный трей** (не закрывается по «крестику»).
- **Глобальный хоткей** `GLOBAL_HOTKEY` (по умолч. `ctrl+alt+j`) — показать/скрыть
  из любого места (нужен `pip install keyboard`).
- **Wake-word** `WAKE_WORD_ENABLED=true` — активация по слову «джарвис» (нужны
  `vosk` + `sounddevice` + модель), окно всплывает и начинает голосовой ввод.

В комплекте готовые плагины:
- `wikipedia` — справки из Википедии (без ключа)
- `jarvis_kb` — RAG-база знаний о самом агенте (без скиллов)
- `vision` — зрение: `analyze_screen` / `analyze_image` (GLM-4V)
- `image_gen` — генерация картинок (CogView-3-Flash)
- `weather` — погода (open-meteo, без ключа)
- `websearch` — поиск в интернете (DuckDuckGo, `pip install ddgs`)

## 👁 Зрение и 🗣 голос

- **Зрение** — плагин `vision` шлёт скриншот в бесплатную `glm-4v-flash`:
  «что на экране?», «есть ли ошибки в коде на экране?».
- **Озвучка (TTS)** — тумблер 🔊 в чате или `TTS_ENABLED=true`; офлайн через
  `pyttsx3` (`pip install pyttsx3`).
- **Голосовой ввод (STT)** — кнопка 🎤 (запись 5 с) через `vosk` офлайн:
  `pip install vosk sounddevice` + модель `vosk-model-small-ru` в `VOSK_MODEL_PATH`.
- **Стриминг** — ответ модели печатается по словам (для GLM); у остальных
  провайдеров текст приходит целиком.

## 🔌 MCP-клиент (опционально)

Jarvis умеет подключаться к внешним MCP-серверам и пробрасывать их инструменты
агенту. Включается `MCP_ENABLED=true`, серверы описываются в `mcp_servers.json`:

```json
{
  "servers": [
    {"name": "filesystem", "command": "npx",
     "args": ["-y", "@modelcontextprotocol/server-filesystem", "C:/work"]}
  ]
}
```

Требует пакет `mcp` (`pip install mcp`). Инструменты регистрируются под именами
`mcp_<server>_<tool>`.

## 💬 Множественные чаты

Слева на странице чата — список диалогов (как в ChatGPT): кнопка «Новый чат»,
переключение между диалогами, удаление. История хранится в SQLite
(`CHATS_DB_PATH`); при переключении контекст восстанавливается в модель.

## 📦 Сборка, установщик и подпись

**1. Собрать exe:**
```bash
build_gui.bat
```
Соберёт `dist\Jarvis\Jarvis.exe` (PyInstaller, windowed, с иконкой). Рядом с exe
кладутся редактируемые `plugins\`, `.env`, `mcp_servers.json` — папку можно
переносить целиком. При запуске как exe агент работает из своей папки.

**2. Собрать установщик** (нужен [Inno Setup 6](https://jrsoftware.org/isdl.php)):
```bash
build_installer.bat
```
Создаёт `installer_out\Jarvis-Setup-<версия>.exe` — установка в пользовательскую
папку (без админ-прав), ярлыки на рабочем столе и в меню Пуск, опциональный
автозапуск, деинсталлятор.

**3. Подписать exe и установщик:**
```powershell
powershell -ExecutionPolicy Bypass -File sign_exe.ps1
```
Создаёт самоподписанный сертификат и подписывает файлы. ⚠️ Самоподпись не
убирает предупреждение SmartScreen у других — для публичного распространения
нужен сертификат УЦ (платно) или бесплатный для OSS (SignPath, Azure Trusted Signing).

**Linux-сборка** (только на Linux — PyInstaller не кросс-платформенный):
```bash
./build_linux.sh   # -> release_out/Jarvis-linux-x64.tar.gz
```

**Android APK** — см. [`mobile/README.md`](mobile/README.md) (Capacitor + Android Studio).

`.github/workflows/release.yml` на пуш тега `vX.Y.Z` автоматически собирает на

## 📱 Мобильное приложение (PWA)

Управляй компьютером с телефона — без сборки APK и магазинов приложений.

**Как включить:**
1. В GUI: **Настройки → Мобильное приложение → «Запустить сервер для телефона»** —
   поднимет сервер и покажет **QR-код**.
2. На телефоне (в той же Wi-Fi сети) отсканируй QR → откроется приложение Jarvis
   в браузере. Введи токен (`AUTH_TOKEN` из настроек).
3. В браузере: **«Добавить на главный экран»** → иконка λ, запускается как
   обычное приложение (PWA, offline-оболочка через service worker).

**Что умеет:** чат с агентом (через WebSocket), просмотр экрана ПК (скриншот).
Сервер можно поднять и отдельно: `python -m jarvis.main` (или `jarvis-server.exe`),
приложение доступно по адресу `http://<IP-ПК>:8000/app/`. Для доступа вне дома —
через Tailscale (см. ниже).

## 🔄 Автообновление

При старте Jarvis проверяет последний релиз на GitHub (`UPDATE_REPO`) и, если
версия новее текущей (`jarvis.__version__`), показывает уведомление в трее и
сообщение в чате со ссылкой на загрузку. Отключается `UPDATE_CHECK=false`.

## 🛡 Безопасность

| Угроза | Защита |
|---|---|
| LLM просит удалить системный файл | `SAFE_ROOT` ограничивает все операции рабочей зоной |
| LLM удаляет без спроса | `delete_file` требует `confirmed=true`, который ставится только после `ask_user` |
| Перехват WebSocket | Bearer-токен + Tailscale (WireGuard под капотом) |
| Случайный `rm -rf` | Path traversal через `..` блокируется в `safety.resolve_safe()` |

**Важно:** не публикуйте `.env` в Git. Файл уже в `.gitignore`.

## 🐛 Отладка

| Симптом | Решение |
|---|---|
| `GEMINI_API_KEY не установлен` | Заполнить `.env`, перезапустить |
| `Путь вне безопасной зоны` | Поправить `SAFE_ROOT` или использовать относительный путь |
| Observer ничего не пишет | Проверить, что `ENABLE_OBSERVER=true`, посмотреть `data/jarvis.log` |
| WebSocket закрывается с 4403 | Неверный `AUTH_TOKEN`, сверить с `.env` |
| pyautogui ничего не делает на ПК | На некоторых антивирусах нужно дать разрешение на эмуляцию ввода |

```
├── bootstrap.py       # инициализация расширений (плагины + MCP)
├── server/            # FastAPI: REST + WebSocket
├── console.py         # REPL для отладки
└── main.py            # точка входа сервера
plugins/               # сами плагины (wikipedia, jarvis_kb, ...)
mcp_servers.json       # конфиг MCP-серверов
tests/                 # pytest
```
